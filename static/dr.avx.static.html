Seamless AVX-512 Execution
on Unsupported Hardware

Dr.avx addresses Generational ISA Fragmentation (GIF). It enables seamless execution of unmodified AVX-512 binaries on hardware that lacks native support, ensuring compatibility across diverse x86-64 generations via dynamic binary translation.

Get StartedView Source

Transparent Execution

Run binaries compiled for AVX-512 on hardware that lacks native support. No recompilation or source code modification required.

Dynamic Rewriting

Translates AVX-512 instructions at runtime into semantically equivalent sequences of supported instructions.

High Performance

Achieves near-native performance on real-world workloads like XZ compression and LLaMa.cpp

Getting Started

Build Dr.avx from source. Supports Linux (Ubuntu 20.04+ tested) on x86-64.

1. Install Dependencies

Ubuntu/Debian
sudo apt-get update && \
sudo apt-get install -y build-essential cmake git \
    libunwind-dev libsnappy-dev liblz4-dev libxxhash-dev

2. Build

For maximum performance
git clone https://github.com/solecnugit/Dr.avx.git
cd dravx
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j"$(nproc)"

3. Run

Use Dr.avx as a prefix launcher for your target binary.

Execution
./build/bin64/dravx -- <program> [args...]
# Example:
# ./build/bin64/dravx -- ./unittests/vadd-512
